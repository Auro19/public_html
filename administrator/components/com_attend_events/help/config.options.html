Configuration Options

The Configuration Options page controls how Attend Events integrates with other Joomla components (see Integration with Other Components), and provides options for customizing the registration process for your specific needs (see Registration Control). Formatting options are also available (see Date and Time Formatting and Front-End Control). Finally, a default set of registration fields can be configured, which will be loaded for any new sessions (see Fields).

Integration with Other Components

Attend Events can be configured to access data stored in the Events and Community Builder components. If these components are not installed on your system, the option to integrate will be disabled (as well as their associated options). For proper operation, you should click the "Apply" button when changing either component's integration state. Since a session and its associated event share a significant amount of common information, options have been provided to limit how much data from the Events component should be displayed on a session's page.

Registration Control

Allow Registrations From
The ability to register for a session may be restricted based on a user's access level. This value selected here determines the value for the "Global" option on a session's details page.
Allow Individual Registrations Only
If this boxed is checked, the built-in registration field "Number of Registrations" will not be shown and its value will be set to one (1).
Allow Full processing
This option determines whether Attend Events should monitor the capacity of sessions for which the "Maximum Number of Registrations" has been set. If this ability is enabled, a session's status will be changed to "Full" when there are no more spaces available, and user's will no longer be able to register for the session.
Allow registered user cancellation
This feature controls whether registered users of the site may cancel their registration.

Front-End Control

These options control how the contents of Attend Events appears in the front end.

Start Listing Sessions
Determines what date/time condition must be met for a session to be listed in the front-end. Depending on the specific settings, a session may be listed outside of its registration period. Under these circumstances, the users will be presented with an error message when they try to register.
Stop Listing Sessions
Determines what date/time condition must be met for a session to be no longer listed in the front-end. Depending on the specific settings, a session may be listed outside of its registration period. Under these circumstances, the users will be presented with an error message when they try to register.
List sessions that are full
If disabled, sessions which are full will not be listed in the front-end automatically.
List sessions regardless of availability
If disabled, only sessions for which the user is capable of registering for will be listed.
Show user number registrations
If enabled, the remaining and total capacity of each session will be displayed when listed.
Display Footer in front-end
If enabled, the Attend Events name and version will be displayed at the bottom of all front-end pages.
Display link to Google Maps
If enabled, the session's venue will be linked to Google Maps, using the venue's location information as a search query.
Display List of "Already Registered" users
If enabled, a list of users who have already registered for the session will be displayed after the registration form. In addition, a comment field will be added at the end of the registration form, so that users can leave a message to be displayed in this list.

Fields

Attend Events has three built-in registration fields: Name, Email Address and Number of People. Any additional information that needs to be collected at registration must be accomplished by creating additional registration fields in the Fields tab of a session's details page. For set-ups where additional fields are required frequently, a default set of additional registration fields can be specified here. To add additional fields, click the "New Field" icon in the top left hand corner. To delete fields, click the red "X" at the left side of the row. Similarly, the order in which fields are displayed can be changed by clicking the blue up and down arrows on each row (these arrows are only visible when re-ordering is possible!). For each field, you must provide a name and select a type from the drop down menu. The list of "Type Specific Parameters" will change depending on the field type you selected. For convenience, the options can be collapsed by clicking the expand/collapse button in the top left corner of each type specific option grouping.

Required
If true, the user must fill-in this field before the registration form can be submitted.
Default
A default value that will appear in the field when the registration form is displayed to the user.
Associated Community Builder Field
If this field of a user's Community Builder profile contains an acceptable value, it is used instead of the form's default.Only available when integrated with the Community Builder component.
Tooltip
The text of a tooltip which can be displayed beside the field in the front end. Leaving this blank means no tooltip will be displayed.
Size
The visible size of the text field.
Maximum Length
The total number of characters that the text field can contain.
Rows/Cols
Specifies the textarea field size, or controls the appearance of the table in which the options of a radio type will be displayed (only columns can be specified -- rows are automatically adjusted, depending on your column specification).
