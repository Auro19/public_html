Session Details

The Session Details page contains fields for specifying all aspects of a session -- mandatory fields are marked with a red asterisk. The upper left-hand portion of the page contains a text field for naming the session, and a WYSIWYG editor for describing it. The upper-right hand area allows the selection of a session host, and the specification of an associated event (if integrated with the Events component). The session's status is displayed also displayed. Depending on configuration options, the a session's status may affect if it is visible in the front-end. However, a session's Published attribute must be true for it to appear in the front-end (regardless of other configuration options). The lower portion of the page contains a tab pane with parameters relating to where the session occurs (see Venue), who can register for it (see Registration), and what information needs to be collected (see Fields).

Registration

The fields on the Registration tab control the duration of the registration period and who can register. Like the General tab, the dates and times on which the registration period begins and ends must be specified. Registration may be open to everyone or restricted to registered users only, or guests only. An optional capacity may be specified. A value of zero (0) indicates an unlimited number of spaces.

Fields

Attend Events has three built-in registration fields: Name, Email Address and Number of People. Any additional information that needs to be collected at registration must be accomplished by creating additional registration fields in the Fields tab. Depending on your Configuration settings, the list may include a default set of fields. To add additional fields, click the "New Field" icon in the top left hand corner. To delete fields, click the red "X" at the left side of the row. Similarly, the order in which fields are displayed can be changed by clicking the blue up and down arrows on each row (these arrows are only visible when re-ordering is possible!). For each field, you must provide a name and select a type from the drop down menu. The list of "Type Specific Parameters" will change depending on the field type you selected. For convenience, the options can be collapsed by clicking the expand/collapse button in the top left corner of each type specific option grouping.

Required
If true, the user must fill-in this field before the registration form can be submitted.
Default
A default value that will appear in the field when the registration form is displayed to the user.
Associated Community Builder Field
If this field of a user's Community Builder profile contains an acceptable value, it is used instead of the form's default.Only available when integrated with the Community Builder component.
Tooltip
The text of a tooltip which can be displayed beside the field in the front end. Leaving this blank means no tooltip will be displayed.
Size
The visible size of the text field.
Maximum Length
The total number of characters that the text field can contain.
Rows/Cols
Specifies the textarea field size, or controls the appearance of the table in which the options of a radio type will be displayed (only columns can be specified -- rows are automatically adjusted, depending on your column specification).

Venue

The Venue tab contains fields for describing where the session takes place. This information can be entered into the fields, or a venue from the Venues Manager can be selected from the Common Venues pull-down menu. If associated with an existing venue, the contents of the session's venue fields are ignored.
